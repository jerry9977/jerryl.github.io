## 1. 人工智能的本质
### 1.1 人工智能的本质：理性决策

课程一开始给出了一些经典定义，例如：

> “让机器完成那些如果由人类完成则需要智能的事情”

但这个定义存在明显问题：

- 什么叫“需要智能”？  
- 人类本身是否总是理性的？  

因此，在这门课中，更强调一种**工程化定义**：

> 人工智能的核心是：在环境中做出合理（rational）的决策

这里的关键不是“智能”，而是：

**在已有信息下，最大化预期结果（performance measure）**

需要特别区分两个概念：

- **Rational（理性）**：在当前信息下做最优选择  
- **Omniscient（全知）**：知道一切（现实中不存在）  

也就是说：

> AI 并不是全知的，而是在不完整信息下尽可能做对

## 2. Agent 建模框架
### 2.1 Agent：统一的建模方式

为了描述“决策”这个过程，课程引入了一个核心抽象：

> Agent（智能体）

一个 Agent 的行为可以表示为：

- 接收环境的 **Observation（观察）**
- 基于当前信息选择 **Action（行动）**

整个流程可以理解为：

Environment → Observation → Agent → Action → Environment


其中最关键的是：

> Agent 的目标是最大化某个 performance measure

例如：

- 自动驾驶：安全性 + 时间  
- 推荐系统：点击率  
- 游戏：得分

## 3. 两种核心范式：Planning 与 Reinforcement Learning

### 3.1 Classical Planning（经典规划）

Planning 是一种**基于模型（model-based）的方法**，它依赖以下强假设：

- 环境是已知的 **(Known Environment / Model)**
- 行动是确定性的 **(Deterministic)**
- 状态完全可观测  **(Fully Observable)**
- 目标是明确的  **(Goal)**

在这种情况下，我们可以：

> 在行动之前就计算出完整的解决方案 (Planning)

形式上表示为：

Initial State → Action Sequence → Goal State

也就是说：

> Planning 的本质是一个搜索问题（search problem）

从 lecture 的角度来看，可以理解为：

> Problem → Planner → Solution

其中 Planner 就是一个通用的求解器（solver）

---

### 3.2 Reinforcement Learning（强化学习）

与 Planning 相反，RL 处理的是：

- 模型未知  
- 环境不确定  
- 无法提前规划  

在这种情况下，Agent 只能：

> 通过与环境交互（interaction）不断学习

RL 的核心机制是：

- 执行动作  
- 接收奖励 **(Reward)**  
- 更新策略 **(Policy)**  

其目标是：

> 最大化长期累计奖励（cumulative reward）

课程中的一个关键假设是：

> Reward Hypothesis：所有目标都可以表示为最大化期望累计奖励

---

### 3.3 Planning vs RL：本质区别

这两种方法的区别可以总结为：

| Planning | Reinforcement Learning |
|----------|----------------------|
| 模型已知 | 模型未知 |
| 使用搜索 | 使用交互 |
| 目标驱动 | 奖励驱动 |
| 行动是确定性 | 行动是随机性 |

## 4. Markov State
### 4.1 Markov Property：关键假设

在强化学习中，一个非常重要的概念是：

> Markov Property（马尔可夫性质）

定义是：

> 未来只依赖当前状态，而不依赖过去

这带来的重要影响是：

- 不需要记录完整历史  
- 当前状态已经包含所有必要信息  

课程中强调：

> 一个满足这一性质的状态称为 Markov State

例如：

- 国际象棋棋盘：是 Markov 状态  
- 股票价格：通常不是  

---

### 4.2 MDP：统一的建模框架

在 Markov 假设下，决策问题可以形式化为：

> Markov Decision Process（MDP）

一个 MDP 包含四个核心要素：

- State（状态）  
- Action（行动）  
- Transition（状态转移）  
- Reward（奖励函数）  

可以表示为：
State + Action → Next State + Reward

MDP 的重要性在于：

> 它统一了 Planning 和 RL 的建模方式
