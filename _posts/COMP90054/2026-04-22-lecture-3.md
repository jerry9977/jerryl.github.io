## 1. 搜索算法（Search Algorithms）

### 1.1 经典规划中的搜索问题
在经典规划中，我们将物理世界抽象为状态空间图：
* **状态 (State)** → 图中的节点
* **动作 (Action)** → 边（有向）
* **规划 (Plan)** → 从 $s_0$ 到 $s_{goal}$ 的路径

> **目标**：在巨大的状态空间中，找到代价最小的最优解。

### 1.2 搜索基本概念
* **核心函数**：
    * $g(n)$：从起点到当前节点的**实际代价** (Path Cost)。
    * $h(n)$：当前到目标的**估计代价**（Heuristic）。
    * $f(n)$：总代价 (Total Cost)。

### 1.3 复杂度参数
* **b (Branching Factor)**：分支因子。
* **d (Goal Depth)**：目标深度。
* **m (Max Depth)**：搜索树最大深度。

## 2. 无信息搜索（Blind Search）

### 2.1 广度优先搜索（BFS）
* **策略**：按层扩展节点（FIFO 队列），优先探索浅层节点。
* **时间复杂度**：$O(b^d)$
* **空间复杂度**：$O(b^d)$（最坏情况下需保存整层 Frontier）。
* **完备性**：在分支因子 $b$ 有限时完备。
* **最优性**：当所有边代价相同（如 $c=1$）时最优。

### 2.2 深度优先搜索（DFS）
* **策略**：优先向深层扩展（LIFO 栈），一条路径走到底再回溯。
* **时间复杂度**：$O(b^m)$
* **空间复杂度**：$O(bm)$（只需保存当前路径及其分支信息）。
* **完备性**：在存在无限深路径时不完备；在有限空间且有环检测时可完备。
* **最优性**：不保证最优。

### 2.3 迭代加深搜索（IDS）
* **策略**：对深度限制 $1, 2, \dots$ 逐步增加的深度受限 DFS。
* **时间复杂度**：$O(b^d)$
* **空间复杂度**：$O(bd)$。
* **完备性**：在分支因子 $b$ 有限时完备。
* **最优性**：当步长代价相同（$c=1$）时最优。

### 2.4 一致代价搜索（UCS）
* **策略**：每次扩展当前路径代价 $g(n)$ 最小的节点。
* **时间复杂度**：$O(b^{\lceil C^* / \epsilon \rceil})$（$C^*$ 为最优解代价，$\epsilon > 0$ 为最小边代价）。
* **空间复杂度**：与时间复杂度同阶。
* **完备性**：当 $\epsilon > 0$ 时完备。
* **最优性**：当 $\epsilon > 0$ 时保证最优。

## 3. 启发式搜索（Heuristic Search）

### 3.1 启发式函数
* **定义**：$h(s)$ 估计从状态 $s$ 到目标的剩余代价。
* **可采纳性 (Admissible)**：$h(s) \le h^*(s)$，绝不高估真实最优代价。
* **一致性 (Consistent)**：满足 $h(s) \le c(a) + h(s')$。一致性必满足可采纳性。

### 3.2 贪心最佳优先搜索（Greedy Best-First Search）
* **策略**：按 $f(n) = h(n)$ 选择节点。
* **时间/空间复杂度**：最坏均为 $O(b^m)$。
* **完备性**：在启发函数安全且使用去重时，在有限空间内完备。
* **最优性**：不保证最优。


**Greedy Best-First Search**
```python
import heapq

def greedy_best_first_search(start):
    # priority queue 按 h 排序（越小越优先）
    open_pq = []
    heapq.heappush(open_pq, (h(start), start))

    closed = set()  # 记录已访问（去重）

    parent = {start: None}  # 可选：用于回溯路径

    while open_pq:
        _, s = heapq.heappop(open_pq)

        if goal_test(s):
            return reconstruct_path(parent, s)

        if s in closed:
            continue
        closed.add(s)

        for s2, _ in successors(s):
            if s2 not in closed:
                # 关键：把所有候选都放进 open（保留分支）
                heapq.heappush(open_pq, (h(s2), s2))
                if s2 not in parent:
                    parent[s2] = s

    return None  # 失败（在有限图 + 合理去重下通常能结束）
```

要点：

- 用 优先队列（open list / frontier） 保存“所有候选”
- 每次取 全局 h 最小 的节点扩展
- 有 closed，可回溯、可避免重复

### 3.3 A* 搜索
* **策略**：按 $f(n) = g(n) + h(n)$ 选择节点。
* **时间复杂度**：最坏为 $O(b^d)$。
* **空间复杂度**：$O(b^d)$。
* **完备性**：当步长代价有正下界且启发函数安全时完备。
* **最优性**：当 $h$ 可采纳时保证最优；若 $h$ 一致，则图搜索中不需要重新打开节点。

### 3.4. 加权 A*（Weighted A*）
* **策略**：$f(n) = g(n) + W \cdot h(n), W \ge 1$。
* **最优性**：不保证最优。
* **有界次优性**：若 $h$ 可采纳，解代价不超过 $W \cdot C^*$。
* **特点**：增大 $W$ 会更偏向启发式，搜索速度通常快于 A*。


## 4. 局部搜索（Local Search）

### 4.1 爬山算法（Hill Climbing）
* **策略**：只保留当前状态，移动到更优的邻居。
* **时间复杂度**：依赖于地形，最坏 $O(b^m)$。
* **空间复杂度**：$O(1)$ 或 $O(b)$。
* **完备性/最优性**：不完备，易陷入局部最优。

从当前 state 出发，在所有邻居中选择最好的那个（greedy），并且丢掉其他所有可能。所以hill climb 不像greedy search，他没有 open list。




**Hill Climbing**
```python
def hill_climbing(start):
    s = start

    while True:
        if goal_test(s):
            return s  # 或返回路径（需自己记录）

        nbrs = [s2 for s2, _ in successors(s)]
        if not nbrs:
            return s  # 无邻居，停

        # 关键：只在“当前邻居”里选最好的
        s_best = min(nbrs, key=lambda x: h(x))

        # 若没有严格改进，则停止（卡在局部最优/平台）
        if h(s_best) >= h(s):
            return s

        # 丢弃其他分支，直接走过去
        s = s_best
```

要点：

- 没有 open/closed，只保留当前状态
- 只在邻居里选最优（local greedy）
- 一旦没有改进就停（容易卡在 local minima / plateau）

### 4.2 强制爬山（Enforced Hill Climbing）
* **策略**：若局部无改进，则从当前状态做一次 **BFS**，找到第一个 $h$ 值更优的状态再跳转。
* **空间复杂度**：局部 BFS 阶段为临时 $O(b^d)$。

## 5. 宽度搜索（Width-Based Search, IW）

### 5.1 IW(k)
* **策略**：按 BFS 扩展，但仅保留**新颖性 (Novelty)** 不超过 $k$ 的状态。
* **时间/空间复杂度**：$O(n^k)$（$n$ 为原子命题数量）。
* **特点**：利用结构做强剪枝，在规划问题中非常有效。

**定义**
一个 state 的 **fact** 有没有出现过**新的组合**

**举例 IW(1)**
```
state A: {at(A)}
state B: {at(A), visited(B)}
state C: {at(A), visited(B)}  ← 没有新信息, State C 会被剪掉
```


>**Question**
>在 TSP 问题中使用 IW(1)，是否能够保证找到解？
>
>答案：不能保证找到解。
> 因为 IW(1) 只**保留**那些引入**新 fact 的状态**，而在 TSP 中，合法路径可能需要重复访问已经访问过的城市 **(例如 A→B→A)**。此时不会产生新的 fact（如 visited(A) 已经存在），该状态会被剪枝，从而**可能将所有通往目标的有效路径剪掉**，因此不能保证找到解。


**执行流程**
```python
from collections import deque
from itertools import combinations


class IW:
    def __init__(self, k):
        self.k = k
        # 存所有出现过的 fact 组合（按大小分开）
        self.seen = {i: set() for i in range(1, k + 1)}

    def is_novel(self, state):
        """
        state: frozenset of facts
        return True if novelty ≤ k
        """
        facts = list(state)

        for size in range(1, self.k + 1):
            for combo in combinations(facts, size):
                combo = tuple(sorted(combo))
                if combo not in self.seen[size]:
                    self.seen[size].add(combo)
                    return True   # novelty = size

        return False  # 等价于 novelty = k+1 → prune

    def search(self, init_state, is_goal, get_successors):
        """
        init_state: frozenset
        is_goal: function(state) -> bool
        get_successors: function(state) -> list of states
        """
        open_list = deque()
        open_list.append(init_state)

        visited = set()
        visited.add(init_state)

        while open_list:
            state = open_list.popleft()

            if is_goal(state):
                return state  # 简化：只返回 goal state

            for succ in get_successors(state):
                if succ in visited:
                    continue

                if self.is_novel(succ):
                    open_list.append(succ)
                    visited.add(succ)
                # else: 被剪掉

        return None
```

- 初始化：open list（BFS）+ initial state + Novelty Table
- 取出 state s（FIFO）
- 扩展 successors s′
- 计算 novelty(s′)（从 1 → k）
- 若 novelty ≤ k：加入 open + 更新 table
- 否则：prune
- 找到 goal 或 open 空 → 结束

## 6. 搜索算法分类总结
* **Blind vs Heuristic**：前者不使用额外信息，后者使用启发函数指导。
* **Systematic vs Local**：前者维护整个 Frontier（完备），后者只关注当前状态（省内存）。
