## 1. Classical Planning Model（经典规划模型）

经典规划（Classical Planning）是 AI 中的一种**基于模型（model-based）的决策方法**，其目标是在已知环境中，通过搜索找到一条从初始状态到目标状态的行动序列。

该模型通常建立在以下假设之上：

- 环境是已知的 **（model known）**
- 状态是完全可观测的 **（fully observable）**
- 行动是确定性的 **（deterministic）**
- 目标是明确的 **（well-defined goal）**

---

### 1.1 State（状态）

状态空间定义为：

$$
S : 所有可能状态的集合
$$

初始状态：

$$
s_0 \in S
$$

目标状态集合：

$$
S_G \subseteq S
$$


- 每一个状态 $s \in S$ 描述了系统在某一时刻的完整信息
- 目标是从 $s_0$ 出发，到达任意一个 $s \in S_G$

---

### 1.2 Action（行动）

行动集合：

$$
A : 所有可能的行动
$$

在某个状态下可执行的行动：

$$
A(s) \subseteq A
$$

一个合法行动满足：

$$
a \in A(s)
$$


- 并非所有行动在所有状态下都可执行
- $A(s)$ 描述当前状态下的可行动作集合（也叫 applicability）

---

### 1.3 Transition Function（状态转移）

状态转移函数定义为：

$$
s' = f(a, s)
$$

- 在状态 $s$ 执行动作 $a$ 后，会得到新的状态 $s'$
- 在 Classical Planning 中，该函数是**确定性的 （deterministic)**

即：

> 相同的 $(s, a)$ 一定产生相同的 $s'$

---

### 1.4 Plan（规划 / 行动序列）

一个规划（plan）定义为一组行动序列：

$$
\pi = [a_0, a_1, ..., a_{n-1}]
$$

从初始状态出发，依次执行这些行动，会得到一条状态轨迹：

$$
s_0 \rightarrow s_1 \rightarrow ... \rightarrow s_n
$$

其中：

$$
s_{i+1} = f(a_i, s_i)
$$

解释：

- plan 是一个**固定的行动序列**
- 在确定性环境下，该序列总是产生相同的执行结果

---

### 1.5 Cost（代价）

规划的总代价定义为：

$$
cost(\pi) = \sum c(a_i, s_i)
$$

解释：

- 每个行动在某个状态下都有一个 cost
- 规划的目标通常是：

> 找到一个使 cost 最小的 plan（optimal planning）





## 2. Markov 相关概念

### 2.1 Markov Property

Markov Property（马尔可夫特性）在已知当前 state 的情况下，未来与过去是条件独立的。

也就是：

$$
P(s_{t+1} | s_t) = P(s_{t+1} | s_0, ..., s_t)
$$

- 只要当前 state 够“完整”
- 过去的信息其实已经被压缩进来了

>Lecturer 也说，其实很多问题本来不是 Markov 的，因为需要 history。但如果我们把需要的 memory（history）一起编码进当前 state，那这个 state 就会变成 Markov state，问题也就可以用 Markov Property 来建模了。


### 2.2 Markov State


一个 Markov state 指的是：

- 该 state 已经包含了所有影响未来决策或状态转移的信息
- 只用这个 state 就可以预测未来, 不需要再看 history


**举例**

Chess（是 **Markov**）

- 当前棋盘 = 全部信息  
- 不需要知道之前怎么走  

股票价格（不是 **Markov**）

- 只看当前 price 不够  
- 缺 trend / history  



### 2.3 Markov Process (or Markov Chain)

Markov Process 就是一个 **state 会随机变化**的系统，每一步怎么变**只取决于当前 state**，不依赖之前发生过什么。

形式就是: 

$$
P(s' | s)
$$

- 有随机性（stochastic）
- 没有 action


### 2.4 Markov Reward Process（MRP）

就是在 Markov Process 的基础上，加了 reward

也就是：

- 有随机性（stochastic）
- 没有 action
- 每个 state 会给一个 reward

形式就是：

$$
P(s' | s)
$$

$$
R(s)
$$

### 2.5 Return

我理解是这样的：

- return 就是一路走下来，这条 **trajectory 拿到的 reward**总和。
- **每个 state 本身会对应一个 reward**，然后你从某个时间点开始，把后面所有 **reward 加起来**，就是这个 **state 的 return**。

$$
G_t = r_{t+1} + r_{t+2} + r_{t+3} + ...
$$


### 2.6 Discount

如果reward一直加：
* 可能无限大
* 而且trajectory不是optimal其实不太重要

所以引入 discount：

$$
G_t = r_{t+1} + γ r_{t+2} + γ² r_{t+3} + ...
$$

我的理解是：

> 越远的 reward，越不重要

γ（gamma）控制这个程度：

* γ = 0 → 只看当前
* γ ≈ 1 → 很看重未来


### 2.7 Value

value 就是平均return

同一个 state，你可能走出很多**不同的 trajectory**，每条 trajectory 都有一个 return
value 就是把这些 **return 全部取平均**

$$
V(s) = E[G_t \mid S_t = s]
$$

## 3. STRIPS and PDDL
### 3.1 什么是 STRIPS？

在 classical planning 里，一个问题通常要描述：

* 有哪些状态
* 初始状态是什么
* 目标状态是什么
* 有哪些动作
* 动作执行后状态怎么变化

课件里说，**STRIPS** 是一种很经典的规划表示模型。一个 STRIPS 问题写成：

$$
P = \langle F, O, I, G \rangle
$$

其中：

* ($F$)：所有 facts / atoms，也就是命题事实集合
* ($O$)：所有 operators，也就是动作集合
* ($I \subseteq F$)：初始状态中为真的事实
* ($G \subseteq F$)：目标中要求为真的事实

每个动作 ($o \in O$) 一般包含三部分：

* ($Pre(o)$)：前提条件，动作能执行前必须为真的 facts
* ($Add(o)$)：执行后会变成真的 facts
* ($Del(o)$)：执行后会被删掉、不再为真的 facts

这就是 STRIPS 的核心：
**state = 一组当前为真的 facts**
**action = precondition + add list + delete list**。


### 3.2 什么是 PDDL？

**PDDL** 全称是 **Planning Domain Definition Language**。
它不是新的规划思想，而是一个标准化写法，用来把 classical planning 问题写成 planner 能读的格式。

* **STRIPS-style semantics** 负责说明动作怎么改变状态
* **PDDL** 是标准语法，用来写 domain file 和 problem file。


#### 3.2.1 PDDL 的两个文件

课件里说，一个 PDDL planning task 通常分成两部分：

#### (1) domain file

描述这个领域里通用的东西：

* predicates
* action schemas

#### (2) problem file

描述某一个具体实例：

* objects
* initial state
* goal state

也就是说：

* **domain** = 规则模板
* **problem** = 具体这次题目的对象、起点、目标

---


## 3.3 STRIPS 的例子

课件用了 blocksworld 举例。比如动作：

### stack(x, y)

含义：把手上拿着的 (x) 放到 (y) 上面。

它的 STRIPS 表示是：

* precondition:

  * holding(x)
  * clear(y)

* add:

  * on(x, y)
  * armEmpty()
  * clear(x)

* delete:

  * holding(x)
  * clear(y)

这个例子正好体现了 STRIPS 三件事：

1. 做动作前要满足前提
2. 做完后有新的事实变真
3. 有些旧事实会失效

比如：

* 原本 holding(x) 为真
* stack 完之后 holding(x) 就不真了
* 所以它进 delete list

课件就是这样给的。

---

## 3.4 PDDL 的例子

同一个动作，用 PDDL 可以写成：

```lisp
(:action stack
  :parameters (?x ?y)
  :precondition (and (clear ?y) (holding ?x))
  :effect (and (arm-empty)
               (on ?x ?y)
               (clear ?x)
               (not (clear ?y))
               (not (holding ?x)))
)
```

这里你可以看到：

* `:parameters` 对应动作参数
* `:precondition` 对应 STRIPS 的 precondition
* `:effect` 里面：

  * 正的 effects 对应 add list
  * `(not ...)` 对应 delete list


## 3.5 PDDL problem file 怎么写

比如 blocksworld 的 problem file 会写：

```lisp
(define (problem bw-abcde)
  (:domain blocksworld)
  (:objects a b c d e)
  (:init
    (on-table a) (clear a)
    (on-table b) (clear b)
    (on-table e) (clear e)
    (on-table c) (on d c) (clear d)
    (arm-empty))
  (:goal
    (and (on e c) (on c a) (on b d)))
)
```

这里：

* `:objects` 是对象集合
* `:init` 是初始状态
* `:goal` 是目标条件

这就是把 STRIPS 里的 (I) 和 (G) 具体写出来。
